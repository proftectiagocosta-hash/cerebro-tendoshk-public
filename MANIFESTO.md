# Manifesto do Cérebro Tendoshk / Manifesto of the Cérebro Tendoshk

[Português](#português) | [English](#english)

---

## Português

O Cérebro Tendoshk nasce de uma necessidade simples e profunda:

**não perder continuidade.**

Conversas se encerram.  
Ideias se dispersam.  
Projetos recomeçam do zero.  
Anotações, documentos, chats, códigos e intenções ficam fragmentados.

Este projeto existe para construir uma ponte entre esses fragmentos.

### O problema

Ferramentas de IA são poderosas, mas frequentemente operam em sessões isoladas.

Quando o contexto se perde, o usuário precisa reconstruir raciocínios, repetir decisões e explicar novamente a própria trajetória.

Isso gera fadiga cognitiva, perda de valor e fragmentação.

### A proposta

O Cérebro Tendoshk propõe uma arquitetura conceitual pública baseada em:

- registros persistentes;
- checkpoints;
- curadoria;
- classificação de contexto;
- separação entre memória bruta e memória útil;
- continuidade entre conversas e projetos;
- proteção de dados sensíveis.

### A visão

A IA não deve ser apenas uma ferramenta de resposta.

Ela também pode se tornar uma camada de organização, reflexão, continuidade e evolução de projetos.

O objetivo não é substituir a mente humana, mas apoiar sua capacidade de lembrar, estruturar, revisar e evoluir.

### Princípio essencial

Nem tudo que é lembrado deve ser público.  
Nem tudo que é público deve revelar o núcleo.

Por isso, este projeto separa:

- material operacional privado;
- documentação pública sanitizada.

### Direção

Este repositório público existe para apresentar a visão, documentar princípios e compartilhar aprendizados sem expor material interno sensível.

O Cérebro Tendoshk é, acima de tudo, uma arquitetura de continuidade.

## English

The Cérebro Tendoshk begins from a simple and deep need:

**to not lose continuity.**

Conversations end.  
Ideas scatter.  
Projects restart from zero.  
Notes, documents, chats, code, and intentions become fragmented.

This project exists to build a bridge between those fragments.

### The problem

AI tools are powerful, but they often operate in isolated sessions.

When context is lost, the user must rebuild reasoning, repeat decisions, and explain the same trajectory again.

That creates cognitive fatigue, loss of value, and fragmentation.

### The proposal

The Cérebro Tendoshk proposes a public conceptual architecture based on:

- persistent records;
- checkpoints;
- curation;
- context classification;
- separation between raw memory and useful memory;
- continuity between conversations and projects;
- protection of sensitive data.

### The vision

AI should not be only a response tool.

It can also become a layer for organization, reflection, continuity, and project evolution.

The goal is not to replace the human mind, but to support its ability to remember, structure, review, and evolve.

### Essential principle

Not everything remembered should be public.  
Not everything public should reveal the core.

For this reason, this project separates:

- private operational material;
- public sanitized documentation.

### Direction

This public repository exists to present the vision, document principles, and share learnings without exposing sensitive internal material.

The Cérebro Tendoshk is, above all, an architecture of continuity.
