# Casos de Uso Públicos / Public Use Cases

[Português](#português) | [English](#english)

---

## Português

Este documento descreve casos de uso seguros, públicos e não sensíveis para o conceito do Cérebro Tendoshk.

### 1. Continuidade de projetos longos

Uma pessoa pode registrar decisões, próximos passos e contexto essencial para retomar um projeto sem reconstruir tudo do zero.

### 2. Organização de estudos

Um estudante pode preservar progresso por tema, bloqueios, referências, dúvidas recorrentes e decisões de método.

### 3. Desenvolvimento criativo

Um criador pode organizar ideias, roteiros, conceitos visuais, rascunhos e decisões de produção ao longo de várias sessões.

### 4. Acompanhamento de projetos técnicos

Um desenvolvedor pode documentar decisões de arquitetura, estados de implementação, problemas, hipóteses e mudanças planejadas.

### 5. Memória de decisões

Um usuário pode registrar por que uma decisão foi tomada, quais alternativas foram rejeitadas, o que motivou a escolha e o que deve ser revisitado depois.

### 6. Retomada entre ambientes

Uma pessoa que trabalha em diferentes máquinas, lugares ou rotinas pode preservar uma base de retomada para não depender apenas da memória imediata.

### 7. Curadoria de conversas

Conversas longas podem ser avaliadas para separar ruído, ideias reaproveitáveis, decisões, pendências e material sensível.

### 8. Separação entre documentação pública e privada

Um projeto pode manter uma visão pública segura enquanto preserva material operacional sensível em ambiente privado.

## English

This document describes safe, public, non-sensitive use cases for the Cérebro Tendoshk concept.

### 1. Long-project continuity

A person can record decisions, next steps, and essential context to resume a project without rebuilding everything from zero.

### 2. Learning organization

A learner can preserve topic progress, blockers, references, recurring questions, and method decisions.

### 3. Creative development

A creator can organize ideas, scripts, visual concepts, drafts, and production decisions across many sessions.

### 4. Technical project tracking

A developer can document architecture decisions, implementation states, issues, hypotheses, and planned changes.

### 5. Decision memory

A user can record why a decision was made, what alternatives were rejected, what motivated the choice, and what should be revisited later.

### 6. Resumption across environments

A person working across different machines, places, or routines can preserve a resumption base instead of relying only on immediate memory.

### 7. Conversation curation

Long conversations can be evaluated to separate noise, reusable ideas, decisions, pending items, and sensitive material.

### 8. Public/private documentation split

A project can maintain a safe public overview while keeping sensitive operational material private.
