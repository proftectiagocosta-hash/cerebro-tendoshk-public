# Casos de Uso Públicos / Public Use Cases

[Português](#português) | [English](#english)

---

## Português

Este documento descreve casos de uso seguros, públicos e não sensíveis para o conceito do Cérebro Tendoshk.

### 1. Continuidade de projetos

Uma pessoa pode usar anotações e checkpoints assistidos por IA para retomar um projeto sem reconstruir o contexto do zero.

### 2. Organização de estudos

Um estudante pode preservar decisões de estudo, progresso por tema, bloqueios, referências e próximos passos.

### 3. Desenvolvimento criativo

Um criador pode organizar ideias, roteiros, conceitos visuais, rascunhos e decisões de produção ao longo de várias sessões.

### 4. Acompanhamento de projetos técnicos

Um desenvolvedor pode documentar decisões de arquitetura, estados de implementação, problemas e mudanças planejadas.

### 5. Memória de decisões

Um usuário pode registrar por que uma decisão foi tomada, quais alternativas foram rejeitadas e o que deve ser revisitado depois.

### 6. Separação entre documentação pública e privada

Um projeto pode manter uma visão pública segura enquanto preserva material operacional sensível em ambiente privado.

## English

This document describes safe, public, non-sensitive use cases for the Cérebro Tendoshk concept.

### 1. Project continuity

A person can use AI-assisted notes and checkpoints to resume a project without rebuilding context from zero.

### 2. Learning organization

A learner can preserve study decisions, topic progress, blockers, references, and next steps.

### 3. Creative development

A creator can organize ideas, scripts, visual concepts, drafts, and production decisions across many sessions.

### 4. Technical project tracking

A developer can document architecture decisions, implementation states, issues, and planned changes.

### 5. Decision memory

A user can record why a decision was made, what alternatives were rejected, and what should be revisited later.

### 6. Public/private documentation split

A project can maintain a safe public overview while keeping sensitive operational material private.
