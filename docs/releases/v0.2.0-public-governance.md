# v0.2.0 — Public Continuity Governance

[Português](#português) | [English](#english)

---

## Português

Esta versão atualiza a vitrine pública do Cérebro Tendoshk para refletir uma maturidade conceitual maior, sem expor o núcleo operacional privado.

### Foco da versão

A versão 0.2.0 reforça publicamente quatro ideias:

1. memória como governança de continuidade;
2. retomada de projetos vivos sem recomeçar do zero;
3. separação rígida entre público e privado;
4. documentação pública como vitrine sanitizada, não como espelho do sistema interno.

### O que mudou

- README atualizado para `0.2.0`.
- Roadmap atualizado com evolução pública controlada.
- Changelog atualizado com registro desta versão.
- Linguagem pública refinada para incluir governança de estado, linhas vivas e verificação.
- Limite público/privado preservado com mais clareza.

### O que continua fora do público

Esta versão não publica:

- prompts privados;
- checkpoints reais;
- conversas privadas;
- logs operacionais;
- masters soberanos internos;
- ferramentas privadas;
- workflows internos sensíveis;
- dados pessoais ou estratégicos.

### Regra de segurança

A camada pública deve explicar, inspirar e orientar conceitualmente.  
Ela não deve permitir reconstrução do núcleo privado nem revelar sua mecânica operacional real.

## English

This release updates the public Cérebro Tendoshk showcase to reflect a more mature conceptual layer without exposing the private operational core.

### Release focus

Version 0.2.0 publicly reinforces four ideas:

1. memory as continuity governance;
2. resumption of living projects without restarting from zero;
3. strict separation between public and private layers;
4. public documentation as a sanitized showcase, not a mirror of the internal system.

### What changed

- README updated to `0.2.0`.
- Roadmap updated with controlled public evolution.
- Changelog updated with this release entry.
- Public language refined to include state governance, living lines, and verification.
- Public/private boundary preserved with greater clarity.

### What remains outside the public layer

This release does not publish:

- private prompts;
- real checkpoints;
- private conversations;
- operational logs;
- internal sovereign masters;
- private tools;
- sensitive internal workflows;
- personal or strategic data.

### Safety rule

The public layer should explain, inspire, and conceptually orient.  
It should not allow reconstruction of the private core or reveal its real operational mechanics.
