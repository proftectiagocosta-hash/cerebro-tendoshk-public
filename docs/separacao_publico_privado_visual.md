# Separação Público/Privado — Modelo Visual / Public-Private Separation — Visual Model

[Português](#português) | [English](#english)

---

## Português

Este documento apresenta um modelo visual público e sanitizado da separação entre a vitrine pública do Cérebro Tendoshk e seu núcleo reservado.

Ele é conceitual. Não descreve processos internos completos nem registros reais.

### Ideia central

```txt
┌───────────────────────────────────────────────┐
│ CAMADA PÚBLICA                                │
│ Visão, manifesto, roadmap, casos de uso,      │
│ exemplos fictícios e linhas vivas sanitizadas │
└───────────────────────────────────────────────┘
                    ↓
┌───────────────────────────────────────────────┐
│ ZONA DE SANITIZAÇÃO                           │
│ Resume, abstrai e transforma material interno │
│ em documentação pública segura                │
└───────────────────────────────────────────────┘
                    ↓
┌───────────────────────────────────────────────┐
│ NÚCLEO RESERVADO                              │
│ Continuidade real, contexto, decisões,        │
│ retomadas, governança e operação privada      │
└───────────────────────────────────────────────┘
```

### Camada pública

A camada pública é a parte visível do projeto. Ela existe para explicar ideias, princípios e modelos sem expor o núcleo reservado.

Exemplos públicos permitidos:

```txt
manifesto
roadmap
arquitetura conceitual
casos de uso públicos
modelos fictícios
linhas vivas sanitizadas
páginas estáticas da vitrine
```

### Zona de sanitização

A zona de sanitização é o filtro entre o que existe internamente e o que pode ser apresentado publicamente.

Ela transforma material interno em documentação segura por meio de:

```txt
resumo
abstração
remoção de detalhes específicos
uso de exemplos fictícios
uso de linguagem conceitual
preservação apenas do aprendizado geral
```

### Núcleo reservado

O núcleo reservado contém a parte operacional, contextual e sensível do Cérebro Tendoshk.

Ele não pertence à vitrine pública.

Exemplos de material reservado:

```txt
registros reais de continuidade
contexto operacional
histórico interno
decisões em andamento
rotinas completas
governança interna detalhada
contextos pessoais
```

### Regra visual simples

```txt
Público = explica o conceito.
Sanitização = transforma o interno em seguro.
Reservado = preserva a operação real.
```

### O que pode virar público

```txt
ideias gerais
modelos fictícios
explicações conceituais
roadmap público
casos de uso sanitizados
lições aprendidas sem detalhes reservados
```

### O que permanece reservado

```txt
registros reais
contextos pessoais
detalhes operacionais completos
material interno de continuidade
rotinas privadas
informações de bastidor
```

### Aplicação no Cérebro Tendoshk

A vitrine pública mostra o que o projeto é, por que ele existe e quais conceitos orientam sua evolução.

O núcleo reservado preserva o que permite a continuidade real do sistema, incluindo contexto, decisões, retomadas e governança interna.

A separação entre os dois é uma escolha de segurança, clareza e maturidade.

## English

This document presents a public, sanitized visual model of the separation between the public Cérebro Tendoshk showcase and its reserved core.

It is conceptual. It does not describe complete internal processes or real records.

### Core idea

```txt
┌───────────────────────────────────────────────┐
│ PUBLIC LAYER                                  │
│ Vision, manifesto, roadmap, use cases,        │
│ fictional examples, sanitized living lines    │
└───────────────────────────────────────────────┘
                    ↓
┌───────────────────────────────────────────────┐
│ SANITIZATION ZONE                             │
│ Summarizes, abstracts, and turns internal     │
│ material into safe public documentation       │
└───────────────────────────────────────────────┘
                    ↓
┌───────────────────────────────────────────────┐
│ RESERVED CORE                                 │
│ Real continuity, context, decisions,          │
│ resumptions, governance, private operation    │
└───────────────────────────────────────────────┘
```

### Public layer

The public layer is the visible part of the project. It exists to explain ideas, principles, and models without exposing the reserved core.

Allowed public examples:

```txt
manifesto
roadmap
conceptual architecture
public use cases
fictional templates
sanitized living lines
static showcase pages
```

### Sanitization zone

The sanitization zone is the filter between what exists internally and what may be shown publicly.

It turns internal material into safe documentation through:

```txt
summary
abstraction
removal of specific details
use of fictional examples
use of conceptual language
preservation of general learning only
```

### Reserved core

The reserved core contains the operational, contextual, and sensitive part of the Cérebro Tendoshk.

It does not belong in the public showcase.

Examples of reserved material:

```txt
real continuity records
operational context
internal history
ongoing decisions
complete routines
detailed internal governance
personal contexts
```

### Simple visual rule

```txt
Public = explains the concept.
Sanitization = turns internal material into safe material.
Reserved = preserves the real operation.
```

### What may become public

```txt
general ideas
fictional models
conceptual explanations
public roadmap
sanitized use cases
lessons learned without reserved details
```

### What remains reserved

```txt
real records
personal contexts
complete operational details
internal continuity material
private routines
behind-the-scenes information
```

### Application in Cérebro Tendoshk

The public showcase shows what the project is, why it exists, and which concepts guide its evolution.

The reserved core preserves what enables the system's real continuity, including context, decisions, resumptions, and internal governance.

The separation between the two is a security, clarity, and maturity choice.
