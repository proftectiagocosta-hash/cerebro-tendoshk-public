# Public Boundaries

This repository exists to explain the project safely.

It must not become a mirror of any private operational system.

## Allowed

- Conceptual explanations
- Generic architecture descriptions
- Fictional templates
- Public roadmap
- Sanitized examples
- Non-sensitive glossary entries
- Security and privacy guidelines

## Not allowed

- Private prompts
- Real checkpoints
- Raw chat exports
- Personal or family history
- Secrets or credentials
- Internal automation logic
- Complete private-system structure
- Private file paths or Drive paths
- Sensitive project history

## Boundary test

Before committing any content, ask:

1. Could this reveal private operating structure?
2. Could this expose personal history?
3. Could this help reconstruct internal protocols?
4. Could this create future security risk?
5. Is this fictional or sanitized enough for public use?

If the answer suggests risk, keep the material out of this repository.
