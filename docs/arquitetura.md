# Arquitetura Conceitual / Conceptual Architecture

[Português](#português) | [English](#english)

## Português

O Cérebro Tendoshk é descrito publicamente como uma arquitetura cognitiva em camadas para preservar continuidade entre conversas, decisões, projetos e retomadas futuras.

Este documento é conceitual. Ele não expõe detalhes de implementação privada.

### 1. Camada de entrada

Onde as informações chegam antes de serem avaliadas:

- conversas;
- anotações;
- documentos;
- ideias;
- decisões;
- registros de projeto;
- dúvidas recorrentes;
- linhas de trabalho em andamento.

### 2. Camada de curadoria

Onde o conteúdo deixa de ser apenas acúmulo e passa por avaliação:

- isso terá valor depois?
- isso registra uma decisão importante?
- isso pertence a um projeto vivo?
- isso deve virar memória, checkpoint, documentação ou descarte?
- isso contém material sensível?
- isso pode ser tornado público com segurança?

### 3. Camada de memória útil

Onde informações relevantes são organizadas para reutilização futura.

Exemplos de categorias públicas e genéricas:

- identidade de projeto;
- decisões estratégicas;
- padrões de trabalho;
- referências importantes;
- próximos passos;
- histórico resumido;
- registros de retomada.

### 4. Camada de continuidade

Onde o sistema ajuda a retomar o trabalho sem recomeçar do zero.

Esta camada existe para preservar o fio entre:

- o que foi decidido;
- o que estava em andamento;
- o que precisa ser retomado;
- o que não deve ser repetido;
- o que ainda precisa amadurecer.

### 5. Camada pública

Onde apenas documentação sanitizada é publicada.

Esta camada funciona como visão geral pública, não como núcleo operacional. Ela comunica a ideia, inspira reutilização conceitual e preserva a fronteira entre o que pode ser mostrado e o que deve permanecer privado.

### Fronteira protegida

O núcleo operacional privado não é espelhado neste repositório. A camada pública deve explicar a arquitetura sem revelar seus mecanismos internos sensíveis.

## English

The Cérebro Tendoshk is publicly described as a layered cognitive architecture for preserving continuity across conversations, decisions, projects, and future resumptions.

This document is conceptual. It does not expose private implementation details.

### 1. Input layer

Where information arrives before being evaluated:

- conversations;
- notes;
- documents;
- ideas;
- decisions;
- project records;
- recurring questions;
- active lines of work.

### 2. Curation layer

Where content stops being mere accumulation and is evaluated:

- will this have future value?
- does this record an important decision?
- does this belong to a living project?
- should this become memory, checkpoint, documentation, or discard?
- does this contain sensitive material?
- can this be made public safely?

### 3. Useful memory layer

Where relevant information is organized for future reuse.

Examples of public and generic categories:

- project identity;
- strategic decisions;
- work patterns;
- important references;
- next steps;
- summarized history;
- resumption records.

### 4. Continuity layer

Where the system helps resume work without restarting from zero.

This layer exists to preserve the thread between:

- what was decided;
- what was in progress;
- what needs to be resumed;
- what should not be repeated;
- what still needs to mature.

### 5. Public layer

Where only sanitized documentation is published.

This layer works as a public overview, not as the operational core. It communicates the idea, inspires conceptual reuse, and preserves the boundary between what can be shown and what must remain private.

### Protected boundary

The private operational core is not mirrored in this repository. The public layer must explain the architecture without revealing sensitive internal mechanisms.
