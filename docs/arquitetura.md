# Arquitetura Conceitual / Conceptual Architecture

[Português](#português) | [English](#english)

---

## Português

O Cérebro Tendoshk é descrito publicamente como uma arquitetura cognitiva em camadas.

Este documento é conceitual. Ele não expõe detalhes de implementação privada.

### 1. Camada de entrada

Onde as informações entram no sistema:

- conversas;
- anotações;
- documentos;
- ideias;
- decisões;
- registros de projeto.

### 2. Camada de curadoria

Onde o conteúdo é avaliado:

- o que é relevante;
- o que deve ser descartado;
- o que deve virar memória;
- o que deve virar checkpoint;
- o que deve virar documentação.

### 3. Camada de memória

Onde informações úteis são organizadas para reutilização futura.

Exemplos de categorias de memória:

- identidade do projeto;
- projetos ativos;
- decisões estratégicas;
- padrões operacionais;
- checkpoints;
- referências importantes.

### 4. Camada de continuidade

Onde o sistema ajuda a retomar o trabalho sem recomeçar do zero.

Exemplos:

- estado atual do projeto;
- próxima ação recomendada;
- breve histórico;
- decisões anteriores;
- pendências.

### 5. Camada pública

Onde apenas documentação sanitizada é publicada.

Esta camada funciona como visão geral pública, não como núcleo operacional.

## English

The Cérebro Tendoshk is described publicly as a layered cognitive architecture.

This document is conceptual. It does not expose private implementation details.

### 1. Input layer

Where information enters the system:

- conversations;
- notes;
- documents;
- ideas;
- decisions;
- project records.

### 2. Curation layer

Where content is evaluated:

- what is relevant;
- what should be discarded;
- what should become memory;
- what should become a checkpoint;
- what should become documentation.

### 3. Memory layer

Where useful information is organized for future reuse.

Examples of memory categories:

- project identity;
- active projects;
- strategic decisions;
- operating patterns;
- checkpoints;
- important references.

### 4. Continuity layer

Where the system helps resume work without restarting from zero.

Examples:

- current project state;
- recommended next action;
- brief history;
- previous decisions;
- pending items.

### 5. Public layer

Where only sanitized documentation is published.

This layer works as a public overview, not as the operational core.
