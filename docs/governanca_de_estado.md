# Governança de Estado / State Governance

[Português](#português) | [English](#english)

---

## Português

A **governança de estado** é um princípio público do Cérebro Tendoshk para evitar que ideias, decisões e projetos vivos sejam tratados como se estivessem todos no mesmo nível de maturidade.

Nem tudo que aparece em uma conversa está pronto. Algumas coisas são apenas hipótese. Outras são decisão. Outras são registro histórico. Outras ainda são caminhos descartados, pendentes ou retomáveis.

A governança de estado existe para preservar essa diferença.

### Por que isso importa

Projetos longos sofrem quando:

- uma ideia inicial é confundida com decisão final;
- um rascunho é tratado como documento aprovado;
- um experimento é confundido com rotina;
- uma intenção futura é confundida com execução realizada;
- um registro antigo continua parecendo atual;
- uma decisão importante fica sem trilha de retomada.

A governança de estado reduz esse risco criando linguagem de acompanhamento.

### Estados públicos conceituais

| Estado | Sentido público |
|---|---|
| Proposto | Ideia levantada, ainda sem validação. |
| Preparado | Estrutura pronta para avaliação ou execução futura. |
| Registrado | Informação preservada para referência. |
| Validado | Conteúdo conferido e aceito como correto naquele contexto. |
| Retomável | Linha de trabalho pode ser retomada sem recomeçar do zero. |
| Pendente | Requer decisão, revisão ou ação futura. |
| Supersedido | Foi substituído por versão mais atual. |
| Descartado | Mantido apenas como histórico ou removido da linha ativa. |

### Exemplo simples

Um projeto pode passar por uma sequência como:

```txt
Proposto → Preparado → Validado → Retomável
```

Mas também pode seguir outro caminho:

```txt
Proposto → Pendente → Supersedido
```

O valor está em não confundir esses estados.

### Relação com memória

Neste projeto, memória não é apenas acúmulo. Memória útil precisa indicar:

- o que foi decidido;
- o que ainda é hipótese;
- o que foi validado;
- o que está pendente;
- o que pode ser retomado;
- o que não deve ser reaberto sem motivo.

### Limite público

Esta página explica o conceito de forma pública e sanitizada. Ela não descreve mecanismos privados, rotinas internas, arquivos reservados, checkpoints reais ou processos operacionais completos.

## English

**State governance** is a public principle of the Cérebro Tendoshk designed to prevent ideas, decisions, and living projects from being treated as if they all had the same level of maturity.

Not everything that appears in a conversation is ready. Some things are hypotheses. Others are decisions. Others are historical records. Others are discarded, pending, or resumable paths.

State governance exists to preserve those differences.

### Why it matters

Long projects suffer when:

- an early idea is mistaken for a final decision;
- a draft is treated as an approved document;
- an experiment is mistaken for a routine;
- a future intention is mistaken for completed execution;
- an old record keeps looking current;
- an important decision has no resumption trail.

State governance reduces this risk by creating tracking language.

### Public conceptual states

| State | Public meaning |
|---|---|
| Proposed | Idea raised, not yet validated. |
| Prepared | Structure ready for review or future execution. |
| Recorded | Information preserved for reference. |
| Validated | Content checked and accepted as correct in that context. |
| Resumable | Work line can be resumed without starting from zero. |
| Pending | Requires decision, review, or future action. |
| Superseded | Replaced by a more current version. |
| Discarded | Kept only as history or removed from the active line. |

### Simple example

A project may move through a sequence such as:

```txt
Proposed → Prepared → Validated → Resumable
```

But it may also follow another path:

```txt
Proposed → Pending → Superseded
```

The value lies in not confusing these states.

### Relation to memory

In this project, memory is not simple accumulation. Useful memory should indicate:

- what was decided;
- what is still a hypothesis;
- what was validated;
- what is pending;
- what can be resumed;
- what should not be reopened without reason.

### Public boundary

This page explains the concept in a public and sanitized way. It does not describe private mechanisms, internal routines, reserved files, real checkpoints, or complete operational processes.
