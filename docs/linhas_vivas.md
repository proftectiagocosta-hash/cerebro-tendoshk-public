# Linhas Vivas de Projeto / Living Project Lines

[Português](#português) | [English](#english)

---

## Português

Uma **linha viva de projeto** é uma trilha de continuidade que ainda merece ser preservada, mesmo quando não está em execução ativa.

Ela pode nascer de uma conversa, de uma decisão, de uma hipótese, de uma pausa, de um retorno ou de uma ramificação. O ponto central é que ela não deve ser tratada como ruído apenas porque ainda não virou produto final.

### Por que linhas vivas importam

Projetos longos raramente avançam em linha reta. Eles geram ideias laterais, versões intermediárias, decisões parciais, dúvidas, pausas e retornos.

Sem uma forma de preservar essas linhas, muita coisa valiosa se perde entre sessões, ferramentas, ambientes e mudanças de foco.

### Estados públicos de uma linha viva

| Estado | Sentido público |
|---|---|
| Proposta | A linha surgiu como possibilidade. |
| Preparada | A linha recebeu estrutura mínima para futura retomada. |
| Validada | A linha foi revisada e aceita como útil naquele contexto. |
| Pausada | A linha não está ativa, mas não foi descartada. |
| Retomável | A linha possui informação suficiente para continuar depois. |
| Expandida | A linha gerou novos caminhos ou ramificações. |
| Supersedida | A linha foi substituída por uma versão melhor. |
| Arquivada | A linha foi preservada apenas como histórico. |

### Exemplo sanitizado 1 — Estudo

```txt
Linha: Trilha de estudo de IA
Estado: Retomável
Origem: sequência de conversas sobre fundamentos, ferramentas e exercícios
Valor: evita recomeçar a trilha toda vez que o estudo é pausado
Próxima ação pública: revisar tópicos concluídos e definir o próximo módulo
```

### Exemplo sanitizado 2 — Produto

```txt
Linha: Ideia de ferramenta de organização de projetos
Estado: Preparada
Origem: necessidade recorrente de preservar decisões e retomadas
Valor: transforma dispersão em estrutura revisável
Próxima ação pública: criar um modelo fictício de fluxo de continuidade
```

### Exemplo sanitizado 3 — Criação

```txt
Linha: Universo narrativo em desenvolvimento
Estado: Pausada
Origem: conjunto de ideias, personagens e símbolos
Valor: mantém coerência de mundo sem expor conteúdo privado
Próxima ação pública: registrar apenas conceitos gerais e não sensíveis
```

### Exemplo sanitizado 4 — Pesquisa

```txt
Linha: Hipótese conceitual em investigação
Estado: Validada parcialmente
Origem: comparação entre fontes, ideias e perguntas abertas
Valor: separa hipótese, evidência e decisão
Próxima ação pública: listar incertezas e próximos critérios de verificação
```

### Regra pública

Uma linha viva não é promessa de execução. Ela é uma forma de preservar continuidade com maturidade, contexto e limites claros.

### Limite de segurança

Os exemplos acima são fictícios e sanitizados. Eles não representam registros reais, checkpoints privados, conversas privadas ou mecanismos internos completos.

## English

A **living project line** is a continuity trail that still deserves to be preserved, even when it is not currently active.

It may come from a conversation, a decision, a hypothesis, a pause, a return, or a branch. The core idea is that it should not be treated as noise merely because it has not become a final product yet.

### Why living lines matter

Long projects rarely move in a straight line. They generate side ideas, intermediate versions, partial decisions, open questions, pauses, and returns.

Without a way to preserve these lines, valuable work can be lost across sessions, tools, environments, and shifts in focus.

### Public states of a living line

| State | Public meaning |
|---|---|
| Proposed | The line appeared as a possibility. |
| Prepared | The line received minimal structure for future resumption. |
| Validated | The line was reviewed and accepted as useful in that context. |
| Paused | The line is not active, but was not discarded. |
| Resumable | The line has enough information to continue later. |
| Expanded | The line generated new paths or branches. |
| Superseded | The line was replaced by a better version. |
| Archived | The line was preserved only as history. |

### Sanitized example 1 — Study

```txt
Line: AI study path
State: Resumable
Origin: sequence of conversations about fundamentals, tools, and exercises
Value: avoids restarting the whole learning path whenever study pauses
Next public action: review completed topics and define the next module
```

### Sanitized example 2 — Product

```txt
Line: Project organization tool idea
State: Prepared
Origin: recurring need to preserve decisions and resumptions
Value: turns dispersion into reviewable structure
Next public action: create a fictional continuity-flow model
```

### Sanitized example 3 — Creation

```txt
Line: Narrative universe in development
State: Paused
Origin: set of ideas, characters, and symbols
Value: preserves world coherence without exposing private content
Next public action: record only general and non-sensitive concepts
```

### Sanitized example 4 — Research

```txt
Line: Conceptual hypothesis under investigation
State: Partially validated
Origin: comparison between sources, ideas, and open questions
Value: separates hypothesis, evidence, and decision
Next public action: list uncertainties and next verification criteria
```

### Public rule

A living line is not a promise of execution. It is a way to preserve continuity with maturity, context, and clear boundaries.

### Safety boundary

The examples above are fictional and sanitized. They do not represent real records, private checkpoints, private conversations, or complete internal mechanisms.
